# new-review

为用户开始填一份新的面试复盘。

## 执行步骤

1. 从用户的话里抽出 `<公司>` 和 `<轮次>`：
   - 公司：用户提到的公司名（如"字节" / "快手" / "腾讯"），不要带"公司"两字
   - 轮次：默认 `一面`；用户说"二面 / leader 面 / hr 面 / 二进宫"等就用对应词

2. 跑脚本：
   ```bash
   bash wiki/scripts/new_review.sh <公司> <轮次>
   ```

3. 把脚本输出原样给用户。文件路径要可点击（用 markdown 链接）。

4. **不要**自动开始填这个文件——复盘必须用户自己填，AI 不知道用户答了什么。

5. 提醒用户：
   - 30 分钟内填完
   - 答崩的考点 → wiki/索引/高频考点榜.md 自评 -1
   - 算法新真题 → wiki/算法/README.md 加进清单
   - 公司画像 → 把这场记录到 wiki/公司/<公司>.md

## 何时不该用这个 skill

- 用户问"我该怎么复盘"——回答方法，不创建文件
- 用户已经在编辑复盘文件——直接帮编辑就行
- 用户要看复盘示例——指 wiki/复盘/示例-2026-05-04-假想字节复盘.md

## 何时主动建议

如果你看到用户消息提到"刚面完 / 面试结束了 / 答崩了 / 答得不错"等明显刚结束面试的语气，主动问一句"要建一份复盘吗？/new-review 帮你打开模板"。
