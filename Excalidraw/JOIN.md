NLJ 算法执行的逻辑是：从驱动表 t1，一行行地取出 a 的值，再到被驱动表 t2 去做 join。也就是说，对于表 t2 来说，每次都是匹配一个值。这时，MRR 的优势就用不上了。 ^dxHwI2HL

看一行扫一遍被驱动表了。 ^RvDGvPDl

MRR（Multi-Range Read）解决的是回表时随机磁盘读的问题。
正常二级索引回表的过程是：从索引里拿到一个主键 ID → 回表去聚簇索引读完整行 → 再拿下一个 ID → 再回表。
这些 ID 在索引里是按索引列排序的，但对应的主键 ID 是乱序的，所以回表时在磁盘上是随机 I/O，东跳一下西跳一下。
MRR 的做法很简单：先把一批主键 ID 攒起来，排个序，再按顺序回表。 ^oYG8fWhh
