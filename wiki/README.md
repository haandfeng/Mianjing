## 维护规范

1. **wiki 只写有出处的内容**——每个论断尽量挂 `[[raw 文件]]` 双链。
2. **双链格式**：raw 用全路径 `[[字节-今日头条-财经业务/我的字节一面]]`；wiki 内部用裸名 `[[Java并发集合]]`。
3. **更新触发**：raw 新增面经 / 修改 → 增量更新 [[索引/公司索引]] · 对应公司画像 · [[索引/高频考点榜]]（公司频次 +1）。
4. **拒绝复述**：wiki 解答如果在 `后端技术专栏/` 已有，只放一句话 + 双链。
5. **保留出处**：每个观点旁标 "出现于 X 公司 X 面"。
6. **复盘环路**：每场面试结束 30 分钟内 → [[复盘/README]] 填一份 → wiki 自评列下移 → [[索引/高频考点榜]] 自动重排短板优先级。
7. **健康检查**：每次大批量改完跑 `wiki-lint`（断链 / TODO / 主题覆盖度）。

