## 命名约定（2026-05-05 整理后）

- **目录**：单字概念名（`公司/` 不是 `公司画像/`；`追问/` 不是 `项目/`；`算法/` 包含一切算法相关）
- **`索引/` 内文件**：去掉 "索引" 前缀（`公司.md` 不是 `公司索引.md`）
- **wiki 双链**：raw 用全路径 `[[字节-今日头条-财经业务/我的字节一面]]`；wiki 内部用裸名 `[[Java并发集合]]`，需要消歧义时加目录前缀 `[[索引/公司]]`

## 维护规范

1. **wiki 只写有出处的内容**——每个论断尽量挂 raw 文件的 wikilink。
2. **更新触发**：raw 新增面经 → 增量更新 [[索引/公司]] · 对应 [[索引/公司]] 文件 · [[索引/高频考点榜]] 频次。
3. **拒绝复述**：wiki 主题文章 / Agent 答案中如果对方已有，只放一句话 + 双链。
4. **保留出处**：每个观点旁标 "出现于 X 公司 X 面"。
5. **复盘环路**：每场面试结束 30 分钟 → [[复盘/模板]] → 自评下移 → [[索引/高频考点榜]] 优先级重排。
6. **健康检查**：大批量改完跑 `/wiki-lint`（断链 / TODO / 主题覆盖度）。
7. **简历是 ground truth**：项目细节冲突时以 [[我的简历/README]] 为准；新冲突加进 [[我的简历/差异表]]。

## 工作流（一周回环）

```
新面经 → raw                  ← 每场面试后（hook 自动提醒刷新）
   ↓
更新 [[索引/公司]] + [[索引/高频考点榜]]
   ↓
复盘 → 自评 -1                  ← 你填 [[复盘/模板]]
   ↓
[[索引/高频考点榜]] 重排
   ↓
下次面试前先看 [[工作流/考前1周]]
```
