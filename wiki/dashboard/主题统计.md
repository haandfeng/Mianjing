# 主题统计

> 主题文章被引用次数 + raw 来源密度，识别哪些是高频考点 + 哪些是孤岛。

## 全主题（按引用次数排序）

```dataview
TABLE WITHOUT ID
  file.link AS "主题",
  domain AS "域",
  length(file.inlinks) AS "被引用次数",
  length(file.outlinks) AS "引用别人次数"
FROM "wiki/技术主题"
WHERE type = "topic"
SORT length(file.inlinks) DESC
```

## 主题被哪些公司画像引用

```dataview
TABLE WITHOUT ID
  file.link AS "主题",
  filter(file.inlinks, (l) => contains(l.path, "wiki/公司/")) AS "出现于公司"
FROM "wiki/技术主题"
WHERE type = "topic"
LIMIT 20
```

## 孤岛主题（无引用）

```dataview
LIST
FROM "wiki/技术主题"
WHERE type = "topic" AND length(file.inlinks) = 0
```

> 这些主题没被任何公司画像 / 其他主题引用 —— 可能不重要，或双链没挂上。

## 没填 domain 字段的主题

```dataview
LIST
FROM "wiki/技术主题"
WHERE type = "topic" AND !domain
```

> domain 字段缺失会让"按域分组"失真，慢慢补一下。

---

## 用法

- 投资学习时间看 "被引用次数" 倒序——次数高的考点性价比高
- 看 "出现于公司" 决定哪些主题该重点准备
- 孤岛主题 → 检查双链 / 删除 / 暂时不管

## 添加新主题

参考 [[模板/新主题文章]]。frontmatter 必填 `domain`：

```yaml
---
title: <主题名>
type: topic
domain: 数据库 | 缓存 | Java | 中间件 | 网络 | Agent
tags: [...]
---
```

加完后跑 `/wiki-lint` 验证。
