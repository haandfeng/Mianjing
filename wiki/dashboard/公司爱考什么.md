# 公司爱考什么

> 按 tag 频次看每家公司的考察重点。**主要看近一年**。

## 公司 × tag 频次（近一年）

```dataview
TABLE WITHOUT ID
  company AS "公司",
  flat(rows.tags) AS "出现的标签"
FROM "公司"
WHERE type = "面经" AND date >= date(today) - dur(365 days)
GROUP BY company
SORT length(rows) DESC
```

## 全 tag 排行（近一年）

```dataview
TABLE WITHOUT ID
  T AS "考点",
  length(rows) AS "出现次数"
FROM "公司"
WHERE type = "面经" AND date >= date(today) - dur(365 days)
FLATTEN tags AS T
GROUP BY T
SORT length(rows) DESC
```

## 公司画像快速查询

```dataview
TABLE WITHOUT ID
  file.link AS "公司",
  tier AS "层级",
  last_interview AS "最近面经",
  total_rounds AS "总轮次",
  passed AS "通过",
  failed AS "挂"
FROM "wiki/公司"
WHERE type = "company-profile"
SORT last_interview DESC
```

## 我面过的公司 × 通过情况

```dataview
TABLE WITHOUT ID
  company AS "公司",
  length(rows) AS "总场数",
  length(filter(rows, (r) => r.verdict = "通过")) AS "通过",
  length(filter(rows, (r) => r.verdict = "挂")) AS "挂"
FROM "公司"
WHERE type = "面经" AND source = "我面试"
GROUP BY company
SORT length(rows) DESC
```

## 用法

- 想看"字节最近一年都问什么"——看"公司 × tag" 表
- 想看"哪些考点是全市场最高频"——看"全 tag 排行"
- 投简历前看"公司画像快速查询" tier / 难度

## 怎么让 dashboard 变好用

每篇面经的 frontmatter 必填 `tags`，至少 3-5 个，例：
- `[java, 并发集合, 锁]`
- `[mysql, 索引, mvcc]`
- `[redis, 缓存击穿, lua]`
- `[agent, mcp, langchain]`
- `[rag, 混合检索, reranker]`

tag 命名：用小写英文 + 中文混排都 OK，但**统一**——比如 `mysql` 不要混 `MySQL`。
