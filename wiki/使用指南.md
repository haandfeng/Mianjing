## Part 0 — Obsidian 基本操作（5 分钟看完）

| 操作 | 快捷键 | 说明 |
|---|---|---|
| 打开任意文件（按名字搜）| `Cmd + O` | 输 `主题`、`快手`、`算法/README` 都能跳 |
| 全文搜索（关键词）| `Cmd + Shift + F` | 找散落在多个文件里的提法 |
| 跳转 wikilink | `Cmd + 点击` 双链 | 点进 `[` `[X]` `]` 跳到 X 文件 |
| 反向看谁引用本文 | 右侧栏 "Backlinks" 面板 | 知道这页被哪些文件提到 |
| 看整个 vault 的连通图 | `Cmd + G` | 看 wiki 拓扑（也能识别孤岛文件）|
| 后退 / 前进 | `Cmd + [` / `Cmd + ]` | 跟浏览器一样 |
| 多 tab 平铺 | `Cmd + 点击` 文件 | 在新 tab 打开 |
| 渲染 Mermaid | 切预览模式 | `Cmd + E` 切换源/预览 |

> **第一次进来怎么逛**：打开 [[wiki/README]] → 按"入口（按场景）"表里的链接走。看 [[可视化/wiki鸟瞰图]] 一图理解整体结构。

## Part 1 — 使用场景（按时间触发）

### 场景 A：收到面试通知（距面 ≥ 7 天）

1. 打开 [[工作流/考前1周]] 跟着 Day -7 → -1 的清单走
2. 重点：
   - [[索引/高频考点榜]] 优先级 ≥ 30 的主题，每个能讲 3 分钟
   - 对应公司画像（[[索引/公司]] 找）
   - 项目 STAR（[[我的简历/README]] 对应实习）

### 场景 B：明天面试（距面 ≤ 24 小时）

1. 打开 [[工作流/考前1天]]
2. 上午看公司 + 反问 → 下午看项目 + 简历对齐 → 晚上算法预热 → 睡前不学习
3. 关键：[[我的简历/差异表]] 确保口径不互冲

### 场景 C：1 小时内进面试（距面 ≤ 1 小时）

1. 打开 [[工作流/考前1小时]]
2. **打印或截图**这页，面前放着
3. 5 张表（自我介绍 / 项目数字 / 公司画像 / 反问 / 算法模板）扫完 = 60 分钟

### 场景 D：面试刚结束（30 分钟内）

**最重要的一步**——记忆新鲜时填复盘。

```
方式 1（命令行）：
   bash wiki/scripts/new_review.sh 字节 一面

方式 2（Claude Code 里）：
   /new-review 字节 一面
   或自然语言："帮我开个字节一面的复盘"
```

打开生成的 `wiki/复盘/2026-MM-DD-字节-一面.md`，按字段填。

填完后：
- 答崩的考点 → [[索引/高频考点榜]] 自评 -1
- 新算法题 → [[算法/README]] 加进真题清单
- 新真题 → [[公司/字节]] 加一行

### 场景 E：每周日晚上（30 分钟）

打开 [[工作流/周末复盘]]，按 5 步清单走：
1. `python3 wiki/scripts/lint_wiki.py` (或 `/wiki-lint`)
2. 复盘归档（这周新加的复盘文件回看一遍）
3. 自评列刷新（[[索引/高频考点榜]]）
4. Offer 板更新（[[offer]]）
5. raw 增量同步检查

### 场景 F：临时查某个考点

1. `Cmd + O` 输关键词（比如 "MVCC"）→ 直接进 [[MySQL-MVCC与隔离级别]]
2. 或从 [[索引/主题]] 顺着分类找

### 场景 G：突击算法

1. [[算法/README]] —— 真题清单 + 6 大模板
2. 按"复习重点"那 5 条优先

### 场景 H：找 Agent / RAG 答题

1. [[Agent问题答案/README]] —— 43 题分主题
2. 概念骨架：[[Agent设计模式与MCP]] / [[RAG优化全链路]]

## Part 2 — 维护 SOP（保持 wiki 不腐烂）

### 加一份新面经

详见 [[工作流/添加新面经]]，简版：

```
1. raw 文件放对目录
   - 大厂 / 中型 → 已有目录
   - 1-2 篇的小公司 → 面经较少公司/<公司>.md
2. 内容用 双栏对照 风格（参考 快手/大数据平台方向/快手一面面经.md）
3. 同步刷新 wiki:
   - 索引/公司.md (如新增公司)
   - 公司/<公司>.md (加一行)
   - 索引/高频考点榜.md (考点频次 +1)
   - 算法/README.md (如有新真题)
4. 跑 lint
5. commit
```

> hook 自动保护：当 Claude 修改 raw 层时会自动提醒刷新 wiki。手动加文件时按这份 SOP 操作。

### 让 Claude 帮你做事（最常用 5 件）

| 你说 | Claude 会 |
|---|---|
| `/wiki-lint` 或"检查 wiki" | 跑 `python3 wiki/scripts/lint_wiki.py`，报告断链 / TODO / 覆盖度 |
| `/new-review 字节 一面` | 建一份新复盘文件 |
| "加这场字节面经到 wiki" | 按 [[工作流/添加新面经]] 流程，建文件 + 同步刷新 |
| "明天面腾讯，给我列复习单" | 读 [[公司/腾讯]] + 高频榜 + 你的自评，输出定制清单 |
| "我刚答崩了 X，帮我改 wiki" | 在主题文章易错点段补一行 + 自评 -1 |

### 自评列怎么改

[[索引/高频考点榜]] 表里"自评"列一栏一栏改：

| 触发 | 怎么动 |
|---|---|
| 这场答崩 | 该考点自评 -1 |
| 这场答得很好 + 面试官认可 | +1（封顶 5）|
| 一周没复习 | -1（避免假信号）|
| 整理完一个新主题 | +1 |

公式 `频次 × 难度 × (6 − 自评)` 自动算优先级，下次打开榜单短板会浮上来。

### 健康检查频率

| 何时跑 lint | 命令 |
|---|---|
| 加完新面经 | `/wiki-lint` |
| 大批量改完（10+ 文件）| `/wiki-lint` |
| 每周日 | [[工作流/周末复盘]] 第 1 步 |
| 月底 | 配合检查 [[health/raw去重建议]] |

### 简历更新时

PDF 简历改了 → 同步改 [[我的简历/README]] 对应文件 → 检查 [[我的简历/差异表]] 有没有新冲突 → wiki 项目页（`wiki/追问/*.md`）如有数字变化也同步。

### 如果发现新口径冲突（简历 vs wiki）

[[我的简历/差异表]] 加一行：项目 / 简历版本 / 不一致版本 / 出处 / 面试统一口径。

### offer 怎么填

每收一份新轮次状态 → 打开 [[offer]] → 表格添一行或更新一行。

## Part 3 — 紧急情况

### "wiki 突然乱了"

1. 跑 `/wiki-lint` 看 broken 数
2. 大于 0：`grep -rn "\[\[<broken-name>\]\]" wiki/` 找出处，手动改或喊 Claude
3. 如果是 raw 文件被删 / 改名导致：[[health/raw去重建议]] 是改名档案

### "Claude 的 commit 都带 Co-Authored-By: Claude，我不想要"

让我以后 commit 时不加，存进 memory 就生效（仅未来；历史不动）。

### "我换电脑了"

memory 不跨设备同步（见上一轮聊天）。CLAUDE.md 是仓库内的 + 跨设备走 git。新电脑：
1. clone 仓库
2. 用 Obsidian 打开
3. 第一次喊 Claude 时它会读 CLAUDE.md 自动按规则做事

### 仓库分层

```
raw 层（仓库根目录）          wiki 层（你这里）
─────────────────────       ───────────────────────
快手/, 字节-.../, ...   →   公司/快手.md, 字节.md, ...
后端技术专栏/*.md       →   技术主题/*.md
Agent技术专栏/*.md      →   Agent问题答案/*.md
项目/卡码信息.md        →   追问/*.md（5 个项目 Q&A）
简历_谭演锋.pdf        →   我的简历/*.md（结构化版）
```
