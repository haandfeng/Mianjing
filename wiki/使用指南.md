# 使用指南

> 个人面试 + 学习知识库的 SOP。**只做 4 件事**——本页是这 4 件事的端到端操作。

```
0. 装插件（一次性）
A. 加新面经       ← 最常用
B. 找面经 / 看公司爱考什么
C. 学习专题
D. 加新专题       ← 第二常用
```

---

## 0. 装插件（一次性 5 分钟）

打开 [[插件设置]]，必装 **Dataview**（dashboard 全靠它）。其他选装。

测试：装完打开 [[dashboard/面试历史]]，能看到表格而不是代码块。

---

## A. 加新面经

> 3 种来源 → 同一个 LLM 模板处理。

### A1. 我刚面完，有录音转录文本

```
1. 拿到录音转录文本（讯飞 / Whisper / 任意 STT）
2. 在 Claude Code 里说：
   "用 [[模板/录音转面经]] 整理这段我自己的面试记录"
   公司：字节
   BU：财经
   轮次：一面
   日期：2026-05-15
   结果：挂
   原文：<贴录音转录>
3. Claude 自动生成 公司/字节/财经/字节财经一面_2026-05-15.md
   含完整 frontmatter + 双链回主题文章
4. 跑 /wiki-lint 验证
5. 在对应公司画像 ([[公司/字节]]) 顶部加一行：
   - 一行 wikilink + 一句话主旨（例：`- 字节财经一面_2026-05-15 —— 这场考了 ConcurrentHashMap、范式、Token`）
```

### A2. 从牛客 / 朋友圈 copy 别人的面经

跟 A1 一样，只是换关键参数：
```
"用 [[模板/录音转面经]] 整理这段网上转载的面经"
公司：字节
BU：财经（不知道写"未知"）
轮次：一面
日期：约 2024-08（不知道写"未知"）
结果：未知
来源：网上转载
原文：<贴 copy 的内容>
```

### A3. 自己手敲（少见）

打开 [[模板/新面经原文]]，全选复制 → 新建文件粘贴 → 替换占位符。

### A 收尾（无论哪种来源）

- [ ] 文件放 `公司/<公司>/<BU>/<标题>.md`（BU 不知道就放 `公司/<公司>/`）
- [ ] frontmatter 含：`company` `bu` `round` `date` `verdict`（通过/挂/未知）`source`（我面试/网上转载）`tags`（3-8 个）
- [ ] 公司画像加一行 wikilink
- [ ] 跑 `/wiki-lint`

## B. 找面经 / 看公司爱考什么

### B1. 我下周可能要面 X 公司

```
Cmd+O → 输 "公司/X" 或 X 的名字
↓
打开公司画像 (e.g. [[公司/字节]])
↓
看：
  · 概览（方向 / 难度 / 一句话）
  · 考察重点（出现次数排序）
  · 真题摘录（直接背）
  · 复习清单双链 → 跳到 [[Java并发集合]] 等八股
```

### B2. 想知道整体趋势

打开 [[dashboard/面试历史]]：
- 近一年所有面经的时间线 + 通过状态
- 按公司分组的场数

打开 [[dashboard/公司爱考什么]]：
- 每家公司高频 tag
- 全市场最热考点排行

> 数据是 Dataview 从 frontmatter 自动算的，加完 frontmatter 就自动出现。

### B3. 想找特定考点出处

```
Cmd+Shift+F → 全文搜 "MVCC" / "Lua" / "ConcurrentHashMap"
↓
选最相关的 [[Java并发集合]] / [[MySQL-MVCC与隔离级别]] 等主题文章
↓
文章末尾"真题来源"列出每家公司哪场问过
```

## D. 加新专题

> 一个考点出现在 ≥3 家公司，就值得建专题。

### D1. 加 raw 笔记

```
新建 专栏/后端/<主题>.md  或  专栏/Agent/<主题>.md
↓
随性写笔记（没有格式要求 — 跟现有专栏的写法一致）
```

### D2. 让 Claude 建 wiki 压缩版

```
对 Claude 说：
"用 [[模板/新主题文章]] 建一个新主题：分布式事务"
基于这些 raw 来源：
  - 专栏/后端/分布式事务.md（我刚写的）
  - 公司/字节/财经/字节财经一面.md（提到 2PC）
  - 公司/快手/支付方向/快手日常实习面经.md（提到 TCC）
↓
Claude 自动生成 wiki/技术主题/分布式事务.md
含 高频问法 + 答题骨架 + 易错点 + 双链
```

### D3. 收尾

- [ ] 在 [[索引/主题]] 适当域加一行索引
- [ ] 顺手在 raw 来源面经的对应公司画像加双链
- [ ] 跑 `/wiki-lint`

## 双链规则速记

| 链谁 | 写法 |
|---|---|
| raw 公司面经 | `[[公司/字节/财经/字节财经一面]]` |
| raw 专栏 | `[[专栏/后端/Java并发编程]]` |
| wiki 主题文章 | `[[Java并发集合]]`（裸名）|
| wiki 索引 / 公司画像 | `[[索引/公司]]` / `[[公司/字节]]` |
| 简历 PDF | 不进 wikilink |

## 常见问题

**Q: dashboard 表格全空？**
A: 1) 装 Dataview；2) 已有面经文件没 frontmatter——用 [[模板/录音转面经]] 重新整理一遍，或手动加 yaml。

**Q: 加完文件双链不工作？**
A: 跑 `/wiki-lint` 看具体哪里 broken。一般是路径写错或者拼写不一致。

**Q: 简历 PDF 在哪？**
A: 仓库根 `简历/` 是 symlink → `/Users/ha/Documents/投递文件/`，按 ISO 日期命名。**不进 git**。

**Q: hook 提醒了"记得增量刷新 wiki"是什么意思？**
A: 修改 raw 文件（公司面经 / 专栏）时，`.claude/hooks/wiki_refresh_hint.sh` 会提醒你顺便更新 wiki 的对应公司画像 + tag 索引。

**Q: 算法在哪？**
A: 不在这个仓库。算法有专门的仓库。
