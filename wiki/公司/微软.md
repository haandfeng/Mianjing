# 微软

## 概览
- 方向：Bing 国际站 图像/视频业务（多媒体应用 + Agent 探索；线上 C#，Agent Python）
- 难度：项目深挖 + AI Coding 观念 + 一道中等算法题；外企节奏
- 一句话画像：西门子 RAG 是核心匹配点（76→94、F1 score 评测、LangSmith trace、LangGraph 状态管理）；ARM 这段相关性弱

## 考察重点（按出现次数排序）
1. **RAG 评测体系** —— Correctness / Relevance / Groundedness 三维度；LLM judge 与人工评测 F1 ≈ 0.8 一致性
2. **LangGraph / LangSmith 理解** —— LangGraph 价值在状态管理 + workflow 编排，底层 SDK 自己调；LangSmith trace 链路 + 评测，企业有数据出境风险，建议自部署
3. **AI Coding 三步法** —— Plan → 人 review → 授权执行；不直接 agent 改代码；改完每行可解释
4. **多轮对话上下文压缩** —— 10 轮 summary 一次作为新 prompt；单轮 5K token，10 轮 50K
5. **ARM 多线程调试 STAR** —— merge 新代码引入并发 bug → MySQL TPS 掉到 0 → 自己梳理线程模型 + AI 辅助 → 修复，证明"底层复杂问题不能只靠 AI 自动修复"
6. **C++ 智能指针** —— shared_ptr 引用计数共享所有权 / unique_ptr 独占 / shared_ptr 不等于线程安全
7. **算法题** —— 岛屿数量（BFS 连通域）；BFS/DFS 空间复杂度严格说都可能 O(m*n)，不要随口说 BFS 更低

## 真题摘录（值得背的）

> **76 到 94 是什么指标？怎么确认的？** 出处 [[微软/Bing图像视频业务实习一面面经]]
> 这是内部评测集上的"回答正确性"分数。同时用 F1 score 评估 LLM 评审与人工评审的一致性（按真阳/假阳/真阴/假阴），F1 ≈ 0.8 说明自动评测可落地。指标口径要统一，不要把 accuracy / correctness / F1 混着说。

> **你怎么理解 LangGraph？** 出处 [[微软/Bing图像视频业务实习一面面经]]
> LangGraph 对我最核心价值是状态管理和 workflow 编排，底层数据库/SDK 调用我很多时候直接调原生 SDK 不依赖它的封装。LangChain/LangGraph 适合快速搭原型，但复杂生产系统通常还是要自己控制底层实现。
