## 考察重点（按出现次数排序）
1. **ThreadPoolExecutor** —— 六大参数 + execute 推进顺序 + 4 种拒绝策略适用场景
2. **Spring Boot 必背注解 Top10** —— `@SpringBootApplication`（= SpringBootConfiguration + EnableAutoConfiguration + ComponentScan）、`@RestController`、`@RequestMapping/@GetMapping/@PostMapping`、`@Autowired`、`@Service`、`@Component`、`@Configuration`、`@Bean`、`@Transactional`、`@Scheduled`
3. **MySQL 优化四层** —— SQL 写法（避免 SELECT * / 避免索引列函数 / LIKE 'xxx%' / 避免 N+1）+ 索引设计（最左前缀 / 避免失效 / 联合索引）+ 表结构（精简字段 / 自增主键 / 冷热分表）+ 架构（读写分离 / 分库分表 / Redis 缓存）
4. **回表 + 覆盖索引** —— 二级索引只有索引列 + 主键值，select 其他列要回主键索引
5. **InnoDB 内存参数** —— `innodb_buffer_pool_size`（物理内存 60-80%）/ `innodb_log_buffer_size` / `sort_buffer_size`（per-thread 慎重）/ `join_buffer_size`
6. **TCP/IP 四层 + HTTPS** —— 应用 / 传输 / 网络 / 网络接口；HTTPS 三风险 + 端口 443 + 对称非对称组合 + TLS 握手
7. **微服务全家桶** —— Nacos 注册发现 + Sentinel 限流 + OpenFeign

## 真题摘录（值得背的）

> **任务从 executor.execute() 到落地执行的流程？** 出处 [[公司/其他/昆仑数智/1]]
> 按"核心线程 → 队列 → 非核心线程 → 拒绝策略"推进：①线程数 < corePoolSize → 直接新建核心线程执行；②核心已满 → 任务入队等待；③队列已满且线程数 < maximumPoolSize → 新建非核心线程；④线程和队列都满 → 触发 RejectedExecutionHandler。

> **什么是回表？怎么减少？** 出处 [[公司/其他/昆仑数智/1]]
> 二级索引叶子节点只有索引列 + 主键值。当查询列不在索引中，需要先在二级索引拿主键，再回主键索引读完整行，多一次磁盘 IO。减少：建覆盖索引（把要查的列也放索引里）或直接按主键查。
