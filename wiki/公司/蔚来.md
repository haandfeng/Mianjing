## 考察重点（按出现次数排序）
1. **RAG 工程取舍** —— 为什么固定调用而不是工具选择性调用（医疗准确性 > 效率）；温度 = 0 防幻觉；ES → Milvus 因为 4096 维 + 稀疏向量
2. **混合检索三路** —— 千问 3 dense + BGE-M3 sparse + BM25 关键词 → reranker 重排；BGE-M3 reranker（0.2s）vs 千问 reranker（0.6s）vs RRF（最快但效果差）
3. **ReAct 模式 + OpenManus 设计模式** —— Think → Act → Observe → Loop；模板方法（BaseAgent.run 调子类 step）/ 策略（不同 think/act）/ 继承（BaseAgent → ReActAgent → ToolCallAgent → Manus）/ 组合（tools / mcp_clients / memory）
4. **MCP 协议** —— SSE（HTTP 长连接，跨网络）vs stdio（标准 I/O，本地，自动启动子进程）；客户端实现 + 配置 + Agent 集成
5. **生产 Agent 复杂场景** —— 长期记忆（向量库 + 关系库 + 知识图谱）/ 短期记忆（Redis + 滑动窗口 + 摘要压缩）/ Token 超限处理 / 指数退避重试 / 熔断器三态
6. **LangChain vs LangGraph** —— 链式 vs 图编排；LangGraph 内置 checkpoint 可恢复 / 可回溯 / human-in-the-loop
7. **PG 慢 SQL 三件套** —— `log_min_duration_statement`、`pg_stat_statements`（avg_time/total_time）、`pg_stat_activity` 实时

## 真题摘录（值得背的）

> **为什么用 RAG 而不是把知识库封装成工具让大模型选择性调用？** 出处 [[蔚来/蔚来CICD组AI agent实习生一面面经-详细版]]
> 医疗行业准确性 > 效率：选择性调用可能"应该调用但没调用"产生幻觉，风险太高。我们固定调用，无关问题就说"无法从已有知识获取相关回答"，宁可不答也不能错。温度也设 0，防止文档外内容产生法律风险。

> **OpenManus 的 ReAct 模式怎么实现？** 出处 [[蔚来/蔚来CICD组AI agent实习生一面面经-详细版]]
> ReActAgent 抽象 think() / act()；step() = await think() → if should_act → await act()。BaseAgent.run() 是 while loop 调 step()。ToolCallAgent.think() 调 LLM ask_tool 拿 tool_calls 写 memory；act() 执行 tool 拿结果写 memory（变成下次 think 的 observation）。整个内存是 user → assistant(tool_calls) → tool 这种交替序列。

> **温度 = 0 会不会太不智能？** 出处 [[蔚来/蔚来CICD组AI agent实习生一面面经-详细版]]
> 准确性 > 用户体验，明确拒绝 > 错误回答。智能的地方在"怎么把文档搜出来再总结"这一步，不是文档外回答用户。医疗器械行业法律风险考虑，文档没提到的就不希望回答。
