## 考察重点（按出现次数排序）
1. **RAG 全链路深挖** —— Query Refine 多语言改写、混合检索（dense + sparse + BM25）、reranker、增量 embedding、千问 3 4096 维换 Milvus、文档摘要索引、长文档压缩
2. **AI Coding 方法论** —— 不直接用 agent 改代码：先生成 plan → 自己审阅 → 授权执行 → 改完每行 review；与小批量任务分层验证（property-based / differential testing）
3. **ARM Profiling 系统能力深度** —— off-CPU 原理（schedule out/in 时间戳）、修 5.x → 6.8 内核迁移、inline hook / LD_PRELOAD / eBPF / uprobes / kprobes 是面试官想要听的方向
4. **MCP + 私域知识接入** —— Rules 文件（CLAUDE.md/.cursorrules）/ MCP / RAG 三种方案
5. **Java 八股** —— ConcurrentHashMap CAS + synchronized 锁桶头、CAS 三大缺点（ABA / 自旋 / 单变量）、JDK 动态代理（Proxy + InvocationHandler，只能代理接口）、ThreadLocal 线程私有 map
6. **TCP vs UDP** —— 面向连接 vs 无连接、可靠性 vs 实时性
7. **算法题** —— 判断链表有环（快慢指针）、岛屿数量（BFS/DFS 连通域）

## 真题摘录（值得背的）

> **你怎么科学地用 AI Coding？** 出处 [[小红书/小红书产品工程师一面面经]]
> 三步法：①先让 AI 写 plan（要改哪些文件、怎么改），不直接 agent 改代码；②自己 review 方案、追问不理解的、删掉不需要的；③确认无误后授权执行，改完每行关键代码都要能解释。原因：复杂场景下 agent 模式会改出不符合预期的代码。

> **如果不是和应用一起打包，做成系统级监控工具，你会怎么抓锁竞争？** 出处 [[小红书/小红书产品工程师二面面经]]
> 这道题暴露了短板。补：用户态用 LD_PRELOAD/PLT-GOT 拦截 pthread_mutex_lock 等符号；内核观测用 eBPF + uprobes/kprobes 挂 tracepoint，零侵入抓 futex 等待链。背景知识要补：Linux 锁实现（mutex/自旋锁/futex/阻塞唤醒），inline hook 原理，动态二进制插桩边界。

> **Embedding 模型和 Reranker 模型底层区别？**（贝壳找房同问，但核心点小红书也涉及）
> 不是 BERT vs GPT，而是 Bi-Encoder vs Cross-Encoder。Embedding 把 query 和 doc 分别编码再算余弦——可离线预算适合召回；Reranker 把两者拼接输入做 token 级交叉注意力，输出相关性分数——精度更高但每对都要过模型，只能精排 top-20。
