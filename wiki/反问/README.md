# 反问体系总览

> 反问的本质是双向选择 + 展示对岗位的思考深度。结尾的 5 分钟决定面试官记不记得你。

## 三大类（任意场都可组合）

### 1. 团队 / 业务定位
- 团队在公司内部的定位、上下游
- 业务当前阶段（拓荒 / 增长 / 维稳）
- 实习生主要参与哪一类工作

### 2. 技术栈深度
- 技术栈细节（不是问"用什么语言"，而是"为什么选 X 而不是 Y"）
- 技术挑战与瓶颈
- 你正在做的方向有什么具体落地难点

### 3. 成长路径
- 实习生培养机制
- 转正 / 留任标准
- 对你这场面试表现的反馈

## 反问原则

1. **不问 HR 类问题**（薪资、地点、加班）— 留给 HR 面
2. **不问能 Google 到的问题**（团队规模、产品介绍这类）
3. **问题里要带"你的视角"**，比如"基于刚才你提到的 X，我想问 Y"
4. **二选一不如开放问**：好"为什么没考虑 Z 方案？" 不好"是不是因为 X？"
5. **结尾问反馈**：每场都问，是 wiki [[复盘/README]]环路最重要的输入
