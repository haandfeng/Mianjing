# 复盘机制

> 这是 wiki 反馈环路的核心。没有复盘，wiki 就只是输出。

## 环路

```
面试 → 复盘（30 分钟内）→ 标 wiki gap → 补 wiki → 下一场面试调用补好的内容
```

每场面试结束后 30 分钟内填一份复盘，趁记忆新鲜。

## 三条规则

1. **30 分钟内**：超过 1 小时记忆衰减严重
2. **答崩 / 没准备的题必须挂双链**：链回对应技术专栏文章，下次复习自动覆盖
3. **每条复盘都有"行动项"**：不是写完就结束，而是明确"这周内补到 wiki 哪里"
