# 周末复盘

> 每周日 30 分钟，让 wiki 不变成废墟。

## 1. 跑健康检查（5 分钟）

```bash
cd /Users/ha/Documents/study/面经
python3 wiki/scripts/lint_wiki.py
```

或者 Claude Code 里直接用 `/wiki-lint`。

- [ ] 0 broken links
- [ ] TODO 数量没有失控（≤ 20 ok）
- [ ] 公司覆盖度有没有新增

## 2. 复盘归档（10 分钟）

- [ ] 这周新增的 [[复盘/README]] 文件全部回看一遍
- [ ] 每条"答崩的"项 → 在对应 [[索引/主题]] 主题的"易错点"段补一句
- [ ] 每条"完全没准备到的"项 → 在 [[索引/高频考点榜]] 检查是否需要新建主题文章

## 3. 自评列刷新（5 分钟）

- [ ] 打开 [[索引/高频考点榜]]
- [ ] 这周答崩的考点 → 自评 -1
- [ ] 这周答得好的 → 自评 +1（封顶 5）
- [ ] 优先级会自动重排，下周看哪些升上来了

## 4. Offer 板更新（5 分钟）

- [ ] 打开 [[offer]]
- [ ] 这周新增轮次的状态填进去
- [ ] 决策列权衡（5 条标准 vs 当前 offer）
- [ ] 已挂的 / 已拒的去掉

## 5. raw 增量同步（5 分钟）

- [ ] 这周新加的 raw 面经 → 同步加进 [[索引/公司]]
- [ ] 对应 [[索引/公司]] 文件增加一行
- [ ] 任何被多家公司反复问的考点 → [[索引/高频考点榜]] 频次 +1

> 一般来说，加新面经时 .claude/hooks 已经自动提醒过我增量更新了。这一步是双保险。
