# 添加新面经 SOP

> 你（或 Claude）每场面试结束后，按这份清单把内容沉淀进仓库。约 15 分钟。

## Step 1 — 选放置目录

| 公司类型 | 路径 |
|---|---|
| 大厂主梯队（[字节/快手/腾讯/蚂蚁/阿里云/京东/小红书/微软/蔚来/TT/Shopee 虾皮]）| 已有目录，直接放 |
| 中型公司（[小鹅通 / Cider / PingCAP / 昆仑数智]）| 已有目录，直接放 |
| **新公司**（仓库不存在的）| 在仓库根目录建 `<公司名>/`；如果以后该公司只有 1-2 篇面经，放在 `其他/` 下即可 |
| **不开新目录**：1-2 篇面经的小公司 | 直接放 `其他/<公司名>.md` |

## Step 2 — 文件名约定

```
{我的}<公司><方向>{一面|二面|...|二进宫}.md
```

例子：
- ✅ `我的快手支付一面.md`（亲历）
- ✅ `字节财经一面.md`（亲历）
- ✅ `阿里云消息团队一面面经.md`（亲历）
- ✅ `小红书产品工程师一面面经.md`（亲历）

收的别人面经（牛客 / 朋友转述）用：
- `<公司>{方向}面经.md` 或 数字 `1.md` `2.md`（如 `小鹅通/1.md`）

## Step 3 — 内容结构

参考 [[快手/大数据平台方向/快手一面面经]] 的"当时回答 / 改进后回答"双栏对照风格。最少包含：

```markdown
# 面经记录

**岗位方向：** <技术栈方向>

---

## 1. 自我介绍
（这场怎么讲的，发现什么需要改进）

## 2. <项目/八股深挖>

### Q：<问题>
**当时的回答（较粗糙）：** <实录>
**改进后的回答：** <离线想清楚后的版本>

## N. 反问环节
- Q：...
  → 面试官回答：...

## N+1. 总结 / 自评
- 答崩：...
- 答得好：...
```

如果是收的别人面经（无法填"当时回答"），就只填问题清单 + 你的答题骨架。

## Step 4 — 同步更新（让 wiki 不腐烂）

按 [[README]] 维护规范第 2 条：

- [ ] **[[索引/公司]]** 加新公司？
- [ ] **`wiki/公司/<公司名>.md`** 加一行面经文件 wikilink + 一句话主旨
- [ ] **[[索引/高频考点榜]]** —— 这场出现的考点频次 +1（新考点新增一行）
- [ ] **[[索引/追问]]** —— 如果是项目深挖，对应 `wiki/追问/<项目>.md` 文件加 Q&A
- [ ] **[[算法/README]]** —— 如果有算法题，加进真题清单
- [ ] **[[反问/历史真实反问]]** —— 如果有反问，加一条
- [ ] **复盘** —— 30 分钟内填一份：[[复盘/模板]] 或 `bash wiki/scripts/new_review.sh <公司> <轮次>`
- [ ] **[[我的简历/差异表]]** —— 如果发现简历 vs 你说的口径不一致，加一行

## Step 5 — 验证

跑健康检查：
```bash
python3 wiki/scripts/lint_wiki.py
```
或 Claude Code 里 `/wiki-lint`。

确认：
- [ ] 0 broken links
- [ ] 该公司画像里新加的 wikilink 能跳转

## Step 6 — Commit

按 [[README]] 维护规范第 6 条："约 10 轮对话 commit 一次"。
commit message 格式：

```
add <公司><轮次>面经 + wiki 同步

- raw: <相对路径>
- wiki/索引/公司.md: 新增 X 公司
- wiki/公司/<公司>.md: 新增 1 场
- wiki/索引/高频考点榜.md: <考点 1> 频次 +1, <新考点>
```

---

## 自动化（可选）

`.claude/hooks/wiki_refresh_hint.sh` 已经监听 raw 目录的 Write/Edit。当 Claude 修改 raw 层文件时，会自动输出系统提醒"记得增量刷新 wiki"，等于 Step 4 的备忘。

如果是手动加文件而不是 Claude 加的，hook 不会触发，按这份 SOP 手动来。

## 给 Claude 看的简短版

```
1. raw 文件放对目录（已有目录 / 面经较少公司 / 新建目录）
2. 内容用 快手一面面经.md 的双栏风格
3. 同步刷新：索引/公司 + 公司/X + 索引/高频考点榜 + 索引/追问 + 算法/README + 反问/历史 + 复盘
4. 跑 lint
5. commit
```
