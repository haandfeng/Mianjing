# 西门子实习（2025/04 – 08）

## 简历原文

> 开发并上线基于 **LangChain / LangGraph** 的 **RAG CT 操作问答系统**，现已在西门子医疗西影力学城正式上线服务。

> 搭建 ETL 流水线，从对象存储系统抽取文档并转化至 **Markdown 文档（支持嵌套表格等复杂结构）**，基于 **Milvus** 实现支持混合检索的多向量存储与 **增量嵌入 / 定向删除**，配合**异步任务队列提升约 2 倍嵌入效率**。

> 优化结合的 **稀疏 + 稠密向量 + BM25** 进行混合检索，叠加 **Reranker, RRF, MRR** 算法重排；引入**文档摘要检索** 和 **Query Refine** 等策略，将平均 **Correctness 从 76% 优化到 92%**，**Relevance 与 Groundedness 保持 99%**。

> 基于 **LangSmith** 追踪、评价 RAG 系统，按 **Correctness / Relevance / Groundedness** 三维度对 RAG 系统打分，完善系统可观测性。

## STAR 自陈

- **S**：进组时已有基于 LangChain Chat 的基础 RAG（Elasticsearch + 单链路），上游开源框架停止维护，效果不达标，无评测体系。
- **T**：搭评测流水线 → 重构 ETL/embedding → 提升回答准确率到可上线水平。
- **A**：1) LangSmith 评测流水线 + 200+ 中英文标准问答；2) 重写 ETL（XML→Markdown 嵌套表格 + 增量更新 + 多模型适配）；3) 换 BGE-M3 → 千问3，迁移 ES→Milvus 支持 4096 维；4) 批量 + 异步 embedding 提升吞吐 ×2；5) Query Refine + 三路混合检索（千问3 稠密 + BGE-M3 稀疏 + BM25）+ Reranker（BGE-M3 / RRF）；6) 按机型 metadata filter 隔离不同设备文档。
- **R**：Correctness 76% → 92%，Relevance & Groundedness 维持 99%+，已上线学城。

## 关键数据 / 名词
- **栈**：LangChain / LangGraph / LangSmith / Milvus / 千问3 / BGE-M3 / DeepSeek V3 (judge)
- **数据**：Correctness 76→92%，Relevance/Groundedness 99%+，嵌入效率 ×2，向量维度 4096
- **检索**：千问3 稠密 + BGE-M3 稀疏 + BM25 三路 → Reranker + RRF + MRR
- **评测**：DeepSeek V3 LLM-as-Judge，与人工 F1 ≈ 0.8
- **场景**：西门子医疗 CT 设备操作文档问答，已上线"学城"
