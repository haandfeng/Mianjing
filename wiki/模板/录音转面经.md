# 录音 / Copy → 面经

> 给 LLM 的 prompt 模板。把面试录音转录文本，或者从网上 copy 的面经原文，喂给 LLM，按这份提示词输出标准面经文件。

## Prompt 模板

```
任务：把下面这段「面试原始内容」转换成标准面经文件。

输入类型：[勾一个]
- [ ] 我自己的面试录音转录
- [ ] 网上 copy 的别人的面经
- [ ] 网上 copy 的别人的面经（部分模糊 / 不全）

公司：__（如 字节、快手、腾讯）
BU / 部门：__（如 财经、中国商业化、大数据平台 — 不知道写"未知"）
轮次：__（一面 / 二面 / Leader / HR — 不知道写"未知"）
日期：__（YYYY-MM-DD — 不知道写"约 YYYY-MM"）
我面试结果：__（通过 / 挂 / 未知 — 转载的填"未知"）

【面试原始内容（粘贴在这里）】
···
（你的录音转录或 copy 的面经文本）
···

输出要求：

1. 用以下 markdown frontmatter 开头：
   ---
   title: <标题>
   type: 面经
   company: <公司>
   bu: <BU>
   round: <轮次>
   date: <日期>
   verdict: <通过/挂/未知>
   source: <"我面试" / "网上转载">
   tags: [<根据内容打 3-8 个 tag，如 java, mysql, redis, 并发, 分布式, agent, rag>]
   ---

2. 整理成「双栏对照」风格（参考 公司/快手/大数据平台方向/快手一面面经.md）：
   - 每道题：标题 + **当时的回答（粗糙）** + **改进后的回答**
   - 如果是转载没有"当时的回答"，只填"答题骨架"
   - 答题骨架 3-5 个 bullet，每条 1-2 句

3. 必填章节：
   - # 面经记录
   - **岗位方向：** ...
   - 1. 自我介绍（如有）
   - 2..N 各题（按时间顺序）
   - 反问环节（如有）
   - 总结 / 自评（如有）

4. 提到的考点尽量在末尾用 wikilink 双链回 wiki 主题文章：
   - 出现 MySQL 索引 → `[[MySQL索引与执行计划]]`
   - 出现 Redis 缓存击穿 → `[[Redis缓存三大问题]]`
   - 出现 ConcurrentHashMap → `[[Java并发集合]]`
   - 出现 RAG / Embedding → `[[RAG优化全链路]]`
   - 出现 MCP / Agent 模式 → `[[Agent设计模式与MCP]]`
   - 出现自己的项目（西门子 RAG / 用友 / 黑马点评 / RPC）→ `[[追问/西门子RAG]]` 等

5. 文件保存到 `公司/<公司>/<BU>/<标题>.md`。
```

## 用完后

- [ ] 文件放在对的目录
- [ ] frontmatter 完整（特别是 date / verdict / tags）
- [ ] 在对应 公司/X 画像里加一行：`- 新文件 —— 一句话主旨`
- [ ] 跑 `/wiki-lint` 验证

## 例子（输入示范）

如果你把这段喂给 Claude：

> 公司：字节
> BU：财经
> 轮次：一面
> 日期：2024-08-15
> 我面试结果：挂
> 
> 内容：今天面字节财经一面。一上来自我介绍。然后问 ConcurrentHashMap 1.7 vs 1.8。我说 1.7 是 Segment 分段锁，1.8 是 CAS + synchronized 锁桶头。面试官追问扩容机制，我说 ...

→ Claude 应输出 `公司/字节/财经/字节财经一面_2024-08-15.md`，frontmatter 标 `verdict: 挂` `tags: [java, 并发集合]` 等，正文双栏对照 + 末尾双链 `[[Java并发集合]]`。

## 给 Claude 的速记版

```
读 [[模板/录音转面经]] → 按 prompt 模板格式化用户给的面试内容 → 写到 公司/<X>/<BU>/<标题>.md
```
