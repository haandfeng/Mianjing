比如经典的面试题：已知 `rand5()` 怎么生成 `rand7()`。

要把一个任意范围 $[a, b]$ 的随机数生成器，完美、等概率地映射到另一个范围 $[c, d]$，核心思想需要用到两个技巧：**N进制扩展（扩大范围）** 和 **拒绝采样（保证等概率）**。

1. **归零化**：首先把原函数变成从 $0$ 开始的 `rand(0, N-1)`。
2. **扩大范围 (N进制扩展)**：如果 $N < M$，原函数的范围不够用。我们可以把多次调用的结果看作是“N进制”的各位数字，把它们组合起来扩大范围：`val = val * N + rand(0, N-1)`。只要一直组合，直到组合出的最大状态数 $N^k \ge M$ 即可。
3. **消除偏差 (拒绝采样)**：这是最关键的一步。当你得到一个大于等于 $M$ 的有效范围（比如 $[0, Limit-1]$）时，**不能直接对 $M$ 取模**。因为如果 $Limit$ 不是 $M$ 的整数倍，取模后较小的数字出现的概率会更大。
    - **解决办法**：找到小于等于 $Limit$ 的最大的 $M$ 的整数倍，记为 $MaxLimit$。如果生成的随机数落在了 $[MaxLimit, Limit-1]$ 这个多出来的“尾巴”区间里，我们就**直接丢弃它，重新生成一次**，直到它落在 $[0, MaxLimit-1]$ 之间。然后再取模，就能绝对保证等概率。


```python
import random

# 假设这是你现有的、只能在 [a, b] 范围内等概率取值的黑盒函数
def given_rand(a, b):
    return random.randint(a, b)

def target_rand(a, b, c, d):
    """
    利用 given_rand(a, b) 生成 [c, d] 范围内的等概率随机数
    """
    n = b - a + 1  # 原始函数的总状态数
    m = d - c + 1  # 目标函数的总状态数

    # 步骤1：封装一个生成 0 到 n-1 的基础函数
    def base_rand():
        return given_rand(a, b) - a

    while True:
        # 步骤2：不断调用 base_rand() 扩大范围，直到涵盖目标状态数 m
        val = base_rand()
        current_states = n
        
        while current_states < m:
            val = val * n + base_rand()
            current_states *= n
        
        # 此时，val 也是等概率分布的，范围在 [0, current_states - 1]
        # 步骤3：拒绝采样 (Rejection Sampling)
        # 找到能被 m 整除的最大上限，丢弃尾部的不完整周期，防止取模偏差
        max_valid_limit = (current_states // m) * m
        
        if val < max_valid_limit:
            # 在有效范围内，取模并加上偏移量 c，返回结果
            return (val % m) + c
        
        # 如果 val >= max_valid_limit，直接进入下一轮 while True 重新生成
```



## 逻辑解释

这两个问题直指随机数算法的最核心设计！这段代码的精妙之处全在这里。

我们把这两个问题拆开来看，先说为什么必须从 0 开始，再说这段 `while` 循环到底在干嘛。

### 1. 为什么要先封装：生成 0 到 n-1 的基础函数？

这是为了**完美匹配数学里的“进制拼接”逻辑**。

你可以把生成随机数的过程，想象成“生成一个多位数”。在我们熟悉的十进制中，每一位数字的范围都是 0 到 9，而不是 1 到 10。

如果你想用两个 0 到 9 的随机数拼成一个 0 到 99 的等概率随机数，公式非常自然：

$$结果 = (第一位 \times 10) + 第二位$$

你看，如果第一位是 4，第二位是 2，拼出来就是 42。因为从 0 开始算，没有任何空隙，刚好填满 0 到 99 的这 100 个状态。

但如果你的随机数是从 1 开始的（比如 1 到 $n$），直接相加或相乘会导致结果的分布不均匀，或者中间出现“无法生成的空洞数字”。

把范围平移到 0 到 $n-1$，就像是把每次生成的随机数当成**N进制里的一位独立数字**，这样数学上的拼接才最严密、最公平。

---

### 2. 这段 while 循环怎么理解？

这段代码实现的操作叫做**N进制扩展（N-ary Expansion）**。它解决的问题是：你手头的随机数范围太小了（$n < m$），一次生不出来你想要的结果。

它的核心思想是：**掷一次骰子范围不够，那就多掷几次，把每一次的结果按“N进制”组合成一个更大的数，直到这个数的最大范围能够覆盖你想要的目标范围 $m$。**

让我们逐行拆解这段代码：

- `val = base_rand()`：先摇一次，得到一个基础值。此时 `val` 有 $n$ 种可能（0 到 $n-1$）。
    
- `current_states = n`：记录当前我们一共能生成多少种不同的状态。目前只摇了一次，所以是 $n$ 种。
    
- `while current_states < m:`：如果当前的总状态数还不够覆盖目标状态数 $m$（比如你有 3 个状态，但你需要 7 个），那就进入循环继续摇。
    
- **`val = val * n + base_rand()`：这是最灵魂的一行！** 它相当于**在N进制下，把之前的数字向左移动一位，然后在个位上填入新摇出来的数字**。这样，两个小的随机数就完美合并成了一个大的、绝对等概率的随机数。
    
- `current_states *= n`：因为多摇了一次，组合的可能性翻了 $n$ 倍。
    

---

### 一个直观的例子：抛硬币生成 0 到 4 (用 rand2 生成 rand5)

假设你的 `base_rand()` 相当于抛硬币，只能生成 0 和 1（$n=2$）。你需要生成 0 到 4 的随机数（$m=5$）。

**初始状态：**

- 抛第一次：`val` = 0 或 1。
    
- `current_states` = 2。
    
- 判断：2 < 5，状态不够用，进入循环。
    

**第一轮循环：**

- 再抛一次。把之前的 `val` 乘以 2（这其实就是二进制的左移），加上新抛出的 0 或 1。
    
- 此时 `val` 的可能结果是：00, 01, 10, 11（即十进制的 0, 1, 2, 3）。
    
- `current_states` 更新为 4（能表示 4 个数了）。
    
- 判断：4 依然小于 5，状态还是不够，继续进循环。
    

**第二轮循环：**

- 抛第三次硬币。`val = val * 2 + 新抛出的数字`。
    
- 此时，我们相当于连续抛了三次硬币，能产生 $2 \times 2 \times 2 = 8$ 种等概率的结果（二进制的 000 到 111，也就是十进制的 0 到 7）。
    
- `current_states` 更新为 8。
    
- 判断：8 已经大于 5，退出循环！
    

现在，你就用一个只能生成 0 和 1 的函数，成功造出了一个能等概率生成 0 到 7 的 `val`。拿着这个足够大的范围，再去执行后面的**拒绝采样**（把 5、6、7 丢弃作废，只留 0 到 4），问题就完美解决了。
