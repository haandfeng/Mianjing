## 一、Agent复杂场景处理

### Q1: 如何实现Agent的长期记忆（Long-term Memory）？

**答案**：

长期记忆主要用于存储用户的历史信息、偏好和知识，需要持久化存储。我理解有几种实现方式：

**1. 架构设计**：
- **向量数据库**：存储用户对话历史的语义表示，用于相似对话检索
- **关系数据库**：存储结构化信息（用户ID、偏好、历史记录等）
- **知识图谱**：存储实体关系和知识，支持复杂查询

**2. 具体实现**：
```python
# 伪代码示例
class LongTermMemory:
    def __init__(self):
        self.vector_db = Milvus()  # 存储对话历史embedding
        self.relational_db = PostgreSQL()  # 存储结构化数据
        self.knowledge_graph = Neo4j()  # 存储知识关系
    
    def save_conversation(self, user_id, conversation):
        # 1. 提取关键信息
        key_info = extract_key_info(conversation)
        
        # 2. 生成embedding并存储
        embedding = self.embedding_model.encode(conversation)
        self.vector_db.insert(user_id, embedding, metadata=key_info)
        
        # 3. 更新用户偏好
        preferences = analyze_preferences(conversation)
        self.relational_db.update_user_preferences(user_id, preferences)
        
        # 4. 更新知识图谱
        entities = extract_entities(conversation)
        self.knowledge_graph.update_entities(user_id, entities)
    
    def retrieve_relevant_memory(self, user_id, query):
        # 检索相关历史对话
        query_embedding = self.embedding_model.encode(query)
        relevant_conversations = self.vector_db.search(query_embedding, top_k=5)
        
        # 从知识图谱获取相关实体
        related_entities = self.knowledge_graph.query_related(user_id, query)
        
        return relevant_conversations, related_entities
```

**3. 优化策略**：
- **增量更新**：只存储关键对话，避免存储所有内容
- **定期清理**：删除过时或低价值的历史记录
- **压缩存储**：对历史对话进行摘要，只保留关键信息
- **分层存储**：热数据（最近）用Redis，冷数据（历史）用数据库

**4. 应用场景**：
- 用户偏好学习：记住用户常用的工具、查询模式
- 个性化推荐：基于历史行为推荐相关功能
- 上下文连续性：跨会话保持对话连贯性

### Q2: 如何实现Agent的短期记忆（Short-term Memory）？

**答案**：

短期记忆用于管理当前对话的上下文，需要快速访问和更新。

**1. 实现方式**：
```python
class ShortTermMemory:
    def __init__(self):
        self.redis = Redis()  # 快速访问
        self.context_window = []  # 当前对话上下文
        self.max_tokens = 8000  # 最大token数
    
    def add_message(self, user_id, role, content):
        message = {"role": role, "content": content, "timestamp": time.time()}
        self.context_window.append(message)
        
        # 检查token限制
        current_tokens = self.count_tokens(self.context_window)
        if current_tokens > self.max_tokens:
            self.compress_context()
        
        # 同步到Redis
        self.redis.setex(
            f"context:{user_id}", 
            3600,  # 1小时过期
            json.dumps(self.context_window)
        )
    
    def compress_context(self):
        # 策略1：滑动窗口，保留最近的N条消息
        if len(self.context_window) > 20:
            self.context_window = self.context_window[-20:]
        
        # 策略2：摘要压缩，将旧消息摘要
        if self.count_tokens(self.context_window) > self.max_tokens:
            old_messages = self.context_window[:-10]
            summary = self.summarize(old_messages)
            self.context_window = [{"role": "system", "content": summary}] + self.context_window[-10:]
    
    def get_context(self, user_id):
        # 优先从内存获取
        if self.context_window:
            return self.context_window
        
        # 从Redis恢复
        cached = self.redis.get(f"context:{user_id}")
        if cached:
            return json.loads(cached)
        
        return []
```

**2. 优化策略**：
- **滑动窗口**：只保留最近的N条消息
- **重要性排序**：保留重要的系统消息和用户意图
- **摘要压缩**：将旧消息压缩为摘要
- **分层存储**：关键信息保留完整，次要信息只保留摘要

**3. 与长期记忆的配合**：
- 短期记忆用于当前对话
- 对话结束后，将关键信息提取到长期记忆
- 新对话开始时，从长期记忆加载相关上下文

### Q3: 如何处理大文档（超过模型上下文窗口）？

**答案**：

大文档处理是RAG系统的核心挑战，我理解有几种策略：

**1. 文档分块策略**：
```python
class DocumentProcessor:
    def chunk_document(self, document, max_chunk_size=1000):
        # 策略1：语义分块
        chunks = self.semantic_chunking(document)
        
        # 策略2：层次化分块
        hierarchical_chunks = self.hierarchical_chunking(document)
        
        # 策略3：滑动窗口（重叠）
        sliding_chunks = self.sliding_window(document, overlap=200)
        
        return chunks
    
    def semantic_chunking(self, text):
        # 计算句子embedding
        sentences = split_sentences(text)
        embeddings = [self.embed(s) for s in sentences]
        
        # 计算相似度，在相似度低的地方切分
        chunks = []
        current_chunk = []
        for i, emb in enumerate(embeddings):
            if current_chunk:
                similarity = cosine_similarity(emb, embeddings[i-1])
                if similarity < 0.7:  # 阈值
                    chunks.append(" ".join(current_chunk))
                    current_chunk = []
            current_chunk.append(sentences[i])
        
        return chunks
```

**2. 检索时处理**：
- **多阶段检索**：先粗检索（top 100），再精检索（top 10）
- **重排序**：使用cross-encoder对检索结果重排序
- **相关性过滤**：只返回相关性高的chunk

**3. 生成时处理**：
```python
def handle_large_context(query, retrieved_chunks, max_context=8000):
    # 如果chunks总token数超过限制
    total_tokens = sum(count_tokens(c) for c in retrieved_chunks)
    
    if total_tokens > max_context:
        # 策略1：只保留最相关的chunks
        scored_chunks = score_relevance(query, retrieved_chunks)
        selected_chunks = scored_chunks[:5]  # 只保留top 5
        
        # 策略2：对chunks进行摘要
        if still_too_large(selected_chunks):
            summarized = [summarize_chunk(c) for c in selected_chunks]
            return summarized
        
        return selected_chunks
    
    return retrieved_chunks
```

**4. 增量处理**：
- **流式处理**：边检索边生成，不需要等待所有chunks
- **分页加载**：先加载最相关的，需要时再加载更多
- **摘要预生成**：对长文档预生成摘要，检索时先看摘要

class TokenLimitHandler:
    def __init__(self, max_tokens=8000):
        self.max_tokens = max_tokens
    
    def handle_token_limit(self, context, query):
        current_tokens = self.count_tokens(context)
        
        if current_tokens > self.max_tokens:
            # 策略1：截断最旧的内容
            truncated = self.truncate_old(context)
            
            # 策略2：摘要压缩
            if still_too_large(truncated):

                return compressed

                return key_info
            
            return truncated

**2. API调用失败重试**：
```python
import time
import random

class RetryHandler:
    def __init__(self, max_retries=3, base_delay=1):
        self.max_retries = max_retries
        self.base_delay = base_delay
    
    def call_with_retry(self, func, *args, **kwargs):
        for attempt in range(self.max_retries):
            try:
                return func(*args, **kwargs)
            except Exception as e:
                if attempt == self.max_retries - 1:
                    raise
                
                # 指数退避 + Jitter
                delay = self.base_delay * (2 ** attempt)
                jitter = random.uniform(0, 0.1 * delay)
                time.sleep(delay + jitter)
        
        return None
```

**3. 熔断器模式**：
```python
class CircuitBreaker:
    def __init__(self, failure_threshold=5, timeout=60):
        self.failure_count = 0
        self.failure_threshold = failure_threshold
        self.timeout = timeout
        self.state = "CLOSED"  # CLOSED, OPEN, HALF_OPEN
        self.last_failure_time = None
    
    def call(self, func, *args, **kwargs):
        if self.state == "OPEN":
            if time.time() - self.last_failure_time > self.timeout:
                self.state = "HALF_OPEN"
            else:
                raise Exception("Circuit breaker is OPEN")
        
        try:
            result = func(*args, **kwargs)
            if self.state == "HALF_OPEN":
                self.state = "CLOSED"
                self.failure_count = 0
            return result
        except Exception as e:
            self.failure_count += 1
            self.last_failure_time = time.time()
            
            if self.failure_count >= self.failure_threshold:
                self.state = "OPEN"
            
            raise
```

class FallbackStrategy:
    def __init__(self):

## 二、MCP协议

### Q5: MCP协议是什么？它的传输协议有哪些？

**答案**：

**MCP（Model Context Protocol）**是Anthropic提出的一个开放协议，用于标准化AI应用与工具之间的交互。

**1. 核心概念**：
- **MCP Server**：提供工具和资源的服务端
- **MCP Client**：使用工具和资源的客户端（如Claude Desktop）
- **统一接口**：不同厂商的工具通过MCP统一接口暴露

**2. 传输协议类型**：
MCP支持多种传输协议：

- **stdio（标准输入输出）**：
  - 最常用，适合本地进程通信
  - 通过stdin/stdout进行JSON-RPC通信
  - 适合开发调试

- **HTTP/HTTPS**：
  - 适合远程服务
  - 支持RESTful API
  - 可以跨网络调用

- **WebSocket**：
  - 支持双向实时通信
  - 适合需要推送的场景
  - 低延迟

- **SSE（Server-Sent Events）**：
  - 服务器向客户端推送事件
  - 适合流式输出场景

**3. 协议结构**：
```json
// MCP Server返回工具描述
{
  "tools": [
    {
      "name": "search_database",
      "description": "搜索数据库",
      "inputSchema": {
        "type": "object",
        "properties": {
          "query": {"type": "string"}
        }
      }
    }
  ]
}

// Client调用工具
{
  "method": "tools/call",
  "params": {
    "name": "search_database",
    "arguments": {"query": "用户查询"}
  }
}
```

**4. 实际应用**：
- **开发MCP Server**：将现有工具封装为MCP Server
- **配置客户端**：在Claude Desktop等客户端配置MCP Server
- **工具市场**：使用开源MCP工具（文件操作、数据库查询、API调用等）

**5. 优势**：
- **标准化**：统一的工具接口，不同模型都可以使用
- **可扩展**：易于添加新工具
- **解耦**：工具与模型解耦，可以独立开发

### Q6: 如何开发一个MCP Server？

**答案**：

**1. 基本结构**：
```python
import json
import sys
from typing import Any, Dict

class MCPServer:
    def __init__(self):
        self.tools = {}
        self.resources = {}
    
    def register_tool(self, name: str, description: str, handler):
        self.tools[name] = {
            "name": name,
            "description": description,
            "inputSchema": {
                "type": "object",
                "properties": {}
            },
            "handler": handler
        }
    
    def handle_request(self, request: Dict[str, Any]):
        method = request.get("method")
        params = request.get("params", {})
        
        if method == "tools/list":
            return {
                "tools": [
                    {
                        "name": tool["name"],
                        "description": tool["description"],
                        "inputSchema": tool["inputSchema"]
                    }
                    for tool in self.tools.values()
                ]
            }
        
        elif method == "tools/call":
            tool_name = params.get("name")
            arguments = params.get("arguments", {})
            
            if tool_name in self.tools:
                handler = self.tools[tool_name]["handler"]
                result = handler(**arguments)
                return {"content": [{"type": "text", "text": str(result)}]}
        
        return {"error": "Unknown method"}
    
    def run(self):
        # 通过stdio通信
        for line in sys.stdin:
            request = json.loads(line)
            response = self.handle_request(request)
            print(json.dumps(response))
            sys.stdout.flush()

# 使用示例
server = MCPServer()

def search_files(query: str):
    # 实现文件搜索逻辑
    return f"搜索结果: {query}"

server.register_tool("search_files", "搜索文件", search_files)
server.run()
```

### Q8: 如何实现多Agent协作？

**答案**：

**1. 架构设计**：
```python
class MultiAgentSystem:
    def __init__(self):
        self.agents = {
            "planner": PlannerAgent(),
            "executor": ExecutorAgent(),
            "reviewer": ReviewerAgent()
        }
        self.coordinator = CoordinatorAgent()
    
    def execute_task(self, task):
        # 1. Planner规划任务
        plan = self.agents["planner"].plan(task)
        
        # 2. Executor执行
        results = []
        for step in plan:
            result = self.agents["executor"].execute(step)
            results.append(result)
        
        # 3. Reviewer审查
        review = self.agents["reviewer"].review(results)
        
        # 4. Coordinator协调
        if review["approved"]:
            return results
        else:
            # 反馈给Planner重新规划
            return self.execute_task(review["feedback"])
```

**2. 使用LangGraph实现**：
```python
class MultiAgentState(TypedDict):

        "current_agent": "reviewer"
    }

def reviewer_node(state: MultiAgentState):

graph = StateGraph(MultiAgentState)

    }

```

## 五、Agent设计模式

### Q10: 除了ReAct，还有哪些Agent设计模式？

**答案**：

**1. Plan-and-Execute模式**：
- **原理**：先制定完整计划，再逐步执行
- **适用场景**：复杂多步骤任务
- **实现**：
```python
class PlanAndExecuteAgent:
    def plan(self, task):
        # 使用LLM生成详细计划
        plan = self.llm.generate_plan(task)
        return plan
    
    def execute(self, plan):
        results = []
        for step in plan:
            result = self.execute_step(step)
            results.append(result)
            
            # 检查是否需要调整计划
            if self.need_replan(result):
                plan = self.replan(plan, results)
        
        return results
```

**2. Reflexion模式**：
- **原理**：执行后反思，如果失败则重新尝试
- **适用场景**：需要自我纠错的任务
- **实现**：
```python
class ReflexionAgent:
    def execute_with_reflection(self, task, max_iterations=3):
        for i in range(max_iterations):
            # 执行任务
            result = self.execute(task)
            
            # 反思结果
            reflection = self.reflect(result, task)
            
            if reflection["success"]:
                return result
            
            # 基于反思改进
            task = self.improve_task(task, reflection)
        
        return result
```

**3. Tool-Use模式**：
- **原理**：模型选择工具并调用
- **适用场景**：需要调用外部工具
- **实现**：
```python
class ToolUseAgent:
    def __init__(self):
        self.tools = {
            "search": self.search_tool,
            "calculate": self.calculate_tool,
            "write_file": self.write_file_tool
        }
    
    def execute(self, query):
        # 模型选择工具
        tool_choice = self.llm.choose_tool(query, self.tools)
        
        # 调用工具
        result = self.tools[tool_choice["name"]](tool_choice["args"])
        
        # 基于结果生成回答
        answer = self.llm.generate_answer(query, result)
        return answer
```

**4. Multi-Agent模式**：
- **原理**：多个Agent协作完成任务
- **适用场景**：复杂任务需要分工
- **实现**：见Q8答案

**5. Hierarchical Agent模式**：
- **原理**：层次化Agent，高层Agent分解任务给低层Agent
- **适用场景**：大型复杂系统
- **实现**：
```python
class HierarchicalAgent:
    def __init__(self):
        self.manager = ManagerAgent()
        self.workers = [WorkerAgent() for _ in range(5)]
    
    def execute(self, task):
        # Manager分解任务
        subtasks = self.manager.decompose(task)
        
        # 分配给Worker
        results = []
        for subtask, worker in zip(subtasks, self.workers):
            result = worker.execute(subtask)
            results.append(result)
        
        # Manager整合结果
        final_result = self.manager.aggregate(results)
        return final_result
```

**2. 多阶段检索**：
```python
class MultiStageRetrieval:
    def retrieve(self, query, top_k=10):
        # 第一阶段：粗检索（召回更多）
        candidates = self.coarse_retrieval(query, top_k=100)
        
        # 第二阶段：精检索（精确匹配）
        refined = self.fine_retrieval(query, candidates, top_k=20)
        
        # 第三阶段：重排序
        final = self.rerank(query, refined, top_k=top_k)
        
        return final
    
    def rerank(self, query, docs, top_k):
        # 使用cross-encoder重排序
        scores = []
        for doc in docs:
            score = self.cross_encoder.score(query, doc)
            scores.append((score, doc))
        
        scores.sort(reverse=True)
        return [doc for _, doc in scores[:top_k]]
```

class HybridRetrieval:
